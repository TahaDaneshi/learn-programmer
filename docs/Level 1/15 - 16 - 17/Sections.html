HTML5 Semantic Elements

HTML5 introduced semantic elements that clearly describe their meaning to browsers and developers.

Why use semantic HTML?

Semantic HTML improves accessibility, SEO, and code maintainability.

<article>
    <header>
        <h1>Article Title</h1>
    </header>
    <section>
        <p>Article content...</p>
    </section>
    <footer>
        <p>Author info</p>
    </footer>
</article>

React Components

React components let you split the UI into independent, reusable pieces.

Components can be either class-based or functional with hooks.

Functional Component Example 
function Welcome(props) {
    return <h1>Hello, {props.name}</h1>;
}

عناصر معنایی HTML5

HTML5 عناصر معنایی را معرفی کرد که معنای خود را به مرورگرها و توسعه‌دهندگان به وضوح توصیف می‌کنند.

چرا از HTML معنایی استفاده کنیم؟

HTML معنایی دسترسی‌پذیری، سئو و نگهداری کد را بهبود می‌بخشد.

<article>
    <header>
        <h1>عنوان مقاله</h1>
    </header>
    <section>
        <p>محتوای مقاله...</p>
    </section>
    <footer>
        <p>اطلاعات نویسنده</p>
    </footer>
</article>

کامپوننت‌های ریکت

کامپوننت‌های ریکت به شما اجازه می‌دهند رابط کاربری را به بخش‌های مستقل و قابل استفاده مجدد تقسیم کنید.

کامپوننت‌ها می‌توانند مبتنی بر کلاس یا تابعی با هوک‌ها باشند.

مثال کامپوننت تابعی 
function Welcome(props) {
    return <h1>سلام، {props.name}</h1>;
}
